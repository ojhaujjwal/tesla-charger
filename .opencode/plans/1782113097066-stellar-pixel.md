# Plan: Migrate Custom Lint Rules to @ojhaujjwal/effect-backpressure

## Goal
Replace the 22 locally-defined custom oxlint rules (`lint/rules/*`) with the shared `@ojhaujjwal/effect-backpressure` npm package (v0.0.2), using the **strict** config preset.

## Summary of Changes

### 1. `package.json` — Add dependency
- Add `"@ojhaujjwal/effect-backpressure": "^0.0.2"` to devDependencies
- Optionally remove `"@oxlint/plugins": "^1.60.0"` (no longer directly needed — comes transitively via the package)

### 2. `.oxlintrc.json` — Switch from local plugin to shared package
- Change `jsPlugins`: `["./lint/index.ts"]` → `["@ojhaujjwal/effect-backpressure/oxlint"]`
- Add `extends`: `["@ojhaujjwal/effect-backpressure/configs/strict.json"]`
- Remove all 20 `tesla-charger/*` rules from `rules` (inherited from strict preset)
- Rename rules in `overrides` (prefix change + 2 rule renames):
  | Override file | Old rule | New rule |
  |---|---|---|
  | `**/*.test.ts` | `tesla-charger/no-silent-error-swallow` | `effect-lint/no-silent-error-swallow` |
  | `src/setup-tesla-partner-application.ts` | `tesla-charger/no-console` | `effect-lint/no-console` |
  | `src/memory-usage.ts` | `tesla-charger/no-process` | `effect-lint/no-process` |
  | `src/http-server-layer.ts` | `tesla-charger/no-node-imports` | `effect-lint/no-node-imports` |
- Keep the 3 built-in rules unchanged (`no-explicit-any`, `typescript/consistent-type-assertions`, `typescript/no-non-null-assertion`)

### 3. `lint/` directory — Remove entirely
- Delete `lint/index.ts` (plugin entry point, no longer needed)
- Delete `lint/rules/` (all 22 rule files + `import-extensions.ts`)

### 4. Install & verify
- Run `npm install` to install the new package
- Run `npm run lint` to catch violations from 4 new rules:
  - `no-async-await` — bans `async`/`await`/`new Promise()`
  - `no-effect-any` — bans `any` in `Effect<A, E, R>` type params
  - `no-raw-date` — bans `new Date()` / `Date.now()`
  - `no-satisfies` — bans `satisfies` keyword
- Fix lint violations as needed
- Run `npm run ci` for full validation

## Rule Mapping (tesla-charger → effect-backpressure)

19 rules keep the same name (just change prefix from `tesla-charger/` to `effect-lint/`):
- `no-effect-ignore`, `no-effect-catchallcause`, `no-effect-asvoid`, `no-silent-error-swallow`, `no-service-option`, `no-nested-layer-provide`, `no-void-expression`, `prefer-option-from-nullable`, `no-console`, `no-process`, `no-node-imports`, `no-bun-globals`, `no-require`, `no-plain-it-with-effect`, `no-vitest-modifiers`, `no-oxlint-disable`, `no-ts-ignore`, `no-ts-nocheck`, `no-unknown-effect-error`

| Local name | Package name |
|---|---|
| `pipe-max-arguments` | `no-pipe-overload` |
| `no-disable-validation` | `no-schema-disable-validation` |

4 new rules enabled by strict config (may cause new violations):
- `no-effect-any`, `no-async-await`, `no-raw-date`, `no-satisfies`

1 local-only rule removed (was defined but never enabled): `import-extensions`
